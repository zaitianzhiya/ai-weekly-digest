# AI 每周发展摘要 — 事件分类体系

## 主分类

每个事件必须分配到以下分类之一（可多选，最多 3 个）：

| 分类 | 标签 | 典型特征 |
|------|------|---------|
| 基础模型 | `#foundation-model` | LLM/多模态新模型发布、基准测试、能力突破、API 定价 |
| AI Agent | `#ai-agent` | Agent 框架、自主工作流、MCP 生态、多 Agent 协调 |
| 开源生态 | `#open-source` | 开源模型权重、开源框架、社区项目、许可证变更 |
| 政策监管 | `#policy` | AI 立法、监管行动、出口管制、政府行政令、国际标准 |
| 硬件算力 | `#hardware` | AI 芯片、GPU、数据中心、算力基础设施、芯片制造 |
| AI 安全 | `#ai-safety` | 前沿模型风险、安全研究、网络攻防、对齐、可解释性 |
| 企业动态 | `#enterprise` | AI 企业融资、人才流动、收购、IPO、商业策略 |
| 应用落地 | `#application` | 行业 AI 部署、垂直场景、生产级应用、ROI 数据 |

## 影响力评分规则

每个事件基于以下信号自动评估基础分，再人工修正：

### 正面信号（加分项）
- 多个独立来源确认（+0.3 per extra source）
- 涉及头部企业（OpenAI/Anthropic/Google/NVIDIA/Meta/Microsoft/Apple，+0.5）
- 含量化基准测试数据（+0.3）
- 有官方声明或论文支撑（+0.4）
- 对下游生态有直接成本影响（+0.3）

### 负面信号（减分项）
- 单一来源且无法交叉验证（-0.5）
- 纯 PR 发布无明显技术内容（-0.4）
- 与已有事件重复（-0.3）
- 仅影响极窄领域（-0.2）

### 最终加权公式
```
基础分 + 信号调整 = 原始分
原始分 × (0.30 × 颠覆性 + 0.25 × 突破度 + 0.20 × 生态影响 + 0.15 × 时效性 + 0.10 × 关注度) = 最终权重
```

## 事件信息源分层

### Tier 1: 原始数据源（直接采用，但需交叉验证）
- 各 AI 企业官方发布（博客、API 文档、Release Notes）
- 学术预印本（arXiv）
- 各国政府官方公告
- 证券交易所披露文件

### Tier 2: 引用数据源（作为交叉验证，不单独采用）
- 科技媒体（EdgeN、VentureBeat、BBC、36氪 等）
- 行业数据库（Crunchbase、东方财富）
- 开源社区（GitHub、Hugging Face）

### 来源独立性修正
- 同一生态系统内的多个来源不重复计分（取最高分）
- 中文源和英文源交叉确认比单一语种确认加分更高
